LocalTransfer

Application desktop C++17 / SFML de transfert de fichiers peer-to-peer sur réseau local.

Version 1.0 C++17 SFML 2.6.1 CMake 3.20+ Audit 85/100 macOS ↔ Windows

Créée le 2026-04-18 · Dernière mise à jour 2026-04-18

1. Vue d'ensemble

LocalTransfer permet de transférer fichiers et dossiers entre plusieurs machines desktop (Mac et Windows) connectées au même réseau local, sans configuration préalable, sans cloud tiers, et sans clé USB.

Découverte auto

UDP broadcast toutes les 2 s — les appareils apparaissent et disparaissent en temps réel.

Transfert direct

TCP peer-to-peer, jusqu'à 100 Go par fichier ou dossier, sans intermédiaire.

Pairing sécurisé

Code à 4 chiffres affiché des deux côtés. Accord explicite du destinataire requis.

Multi-destinataires

Sélection de plusieurs pairs → envoi en parallèle, chaque session isolée dans son propre thread.

3. Architecture logicielle

Architecture en couches (Clean Architecture allégée) :

UI (SFML)  →  App (Controllers)  →  Domain  →  Infra + Network
CoucheResponsabilitéNamespace
domainEntités pures : Device, FileMeta, TransferRequest, TransferSessionltr::domain
coreTypes, EventBus thread-safe, Logger, helpers formatltr::core
infraConfig, Filesystem, Hash (SHA-256)ltr::infra
networkProtocol, Discovery (UDP), TransferServer (TCP), TransferClientltr::network
appAppState, AppController (orchestrateur)ltr::app
uiTheme, Screens (MainScreen, IncomingOfferScreen), Widgetsltr::ui

Modèle de concurrence

  • UI thread (main) : boucle SFML, rendu, input
  • Discovery : 3 threads (beacon UDP / listener UDP / TTL sweep)
  • TransferServer : 1 thread d'accept() + 1 thread par session entrante
  • TransferClient : 1 thread par envoi sortant

Communication inter-threads : EventBus (mutex + queue de std::variant<Event>) drainé par le thread UI à chaque frame.

4. Protocole réseau

Découverte — UDP 45454

Beacon JSON diffusé en broadcast toutes les 2 secondes :

{
  "proto":    "LTR1",
  "kind":     "HELLO",
  "id":       "uuid-v4",
  "name":     "Mac de Serge",
  "platform": "macOS",
  "tcpPort":  45455
}

Les pairs sont purgés après 6 secondes sans beacon (TTL). Le socket broadcast active SO_BROADCAST via une sous-classe BroadcastUdpSocket (SFML 2.6 ne l'expose pas nativement).

Transfert — TCP 45455

Framing binaire :

+--------+--------+----------+---------+
| magic  | type   |  length  | payload |
| "LTR1" |  u8    |  u32 BE  |  ...    |
|  4 o   |  1 o   |   4 o    |   N o   |
+--------+--------+----------+---------+
CodeTypePayload
0x01OFFERJSON : sessionId, pin, files[]
0x02ACCEPTJSON : sessionId
0x03REJECTJSON : sessionId, reason
0x04FILE_HEADERJSON : index, relativePath, size
0x05FILE_CHUNKBinaire brut (≤ 256 KB)
0x06FILE_ENDJSON : index, sha256 (optionnel)
0x07DONE(vide)
0x08CANCELJSON : reason
0x09ERRORJSON : code, message

Une offre entrante affiche un overlay modal par-dessus le MainScreen, contenant le code PIN, le résumé des fichiers, et les boutons « Refuser / Accepter ».

6. Parcours utilisateur

  1. L'utilisateur lance l'app sur les deux machines
  2. Les appareils apparaissent dans la sidebar (quelques secondes)
  3. Sélection du ou des destinataires (clic sur DeviceListItem)
  4. Clic sur Parcourir → sélection de fichiers ou d'un dossier
  5. Clic sur ENVOYER
  6. Le destinataire voit une modale avec le code PIN et les fichiers
  7. Il Accepte → le transfert démarre
  8. Les deux côtés voient la progression (% / vitesse / ETA)
  9. À la fin : notification et fichiers dans ~/Downloads/LocalTransfer/

7. Build & exécution

Prérequis

  • CMake ≥ 3.20
  • Compilateur C++17 (Clang sur Mac, MSVC 2022 sur Windows)
  • Git
  • Mac : Xcode Command Line Tools (xcode-select --install)
  • Windows : Visual Studio 2022, workload « Desktop C++ »

Commandes

# Build Release
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release -j

# Avec tests
cmake -S . -B build -DLTR_BUILD_TESTS=ON
cmake --build build -j
ctest --test-dir build --output-on-failure

# Lancement
./build/local_transfer           # macOS
build\Release\local_transfer.exe # Windows
Pare-feu : autoriser les ports UDP 45454 et TCP 45455 en entrée/sortie sur le réseau local. Au premier lancement, macOS et Windows afficheront une boîte de dialogue de demande d'accès.

9. Limitations connues (V1)

Assumées et documentées — non bloquantes pour l'usage personnel sur réseau de confiance.
  • 🚫 Pas de drag-and-drop natif (SFML limitation) — sélection via bouton « Parcourir » uniquement
  • 🔓 Pas de chiffrement TLS des données (LAN de confiance uniquement)
  • ↺ Pas de reprise après interruption réseau
  • 📱 Support desktop uniquement — pas de mobile
  • 👥 Deux instances sur une même machine impossible (port UDP 45454 exclusif)
  • ⏱ Arrêt de l'app peut prendre quelques secondes si des transferts sont en cours (lecture TCP bloquante)
  • 🗂 Suppression individuelle d'un fichier dans la liste désactivée en V1 — utiliser « Vider »
